### A1-1 — Durable process identity (F2 graduates to REQUIRED)

log_path, prompt_path, provider, resolved_model, executor_version,
process_start_token. Record `Popen` results durably: write the run row
BEFORE the process spawn where possible, and reconcile after. On daemon
restart, reconstruct the running-process map from the runs table: for each
non-terminal run row, probe (pid alive? start-token matches?) and either

the step. Kill the in-memory-only `_RUNNING` single point of failure
(spawn.py). The Hermes-executor path must get the same run tracking as
codex/claude paths.

### A1-2 — Resource leases
New table per review §2.0.5: resource_leases (worker_slot, port, disk_mb
kinds first). Dispatch acquires a worker_slot lease before spawn; capacity
comes from operator config (recipes.max_workers, default 2), NEVER from
model output or recipe text. Leases carry lease_until and are reaped on
expiry. Fd/port ceilings are enforced by refusing dispatch when no lease is
available — a full board must queue, not spawn.

### A1-3 — Budget unification
_admit() currently reads FACTORY_BOARD_DAY_TOKEN_CEILING from the
environment while selector admission reads validated config — two
inconsistent implementations. Unify: ONE code path reading the validated
daemon config; the env var dies. Config key: recipes.board_day_token_ceiling.

### A1-4 — Unknown usage stays unknown
record_run_end() converts missing token usage to zero, corrupting telemetry
and any future autonomy predicate. Missing usage must persist as NULL, and
every aggregation (costs_rollup, budget checks) must treat NULL as unknown —
budget admission may not treat unknown as free.

### A1-5 — Watchdog subprocess timeouts
watchdog._run_kanban() uses subprocess.run() without a timeout; one hung CLI
invocation stalls every board. Add bounded timeouts (config, default 120s)
+ a timed-out outcome recorded on the monitor. Due monitors must not run
unbounded serially — cap per-tick monitor wall time.

### A1-6 — Reap failures surface
reap paths swallow every exception while completing/blocking kanban tasks.
Route these through the A0 action-intent journal (fresh attempt keys,
probe-before-retry) or, minimally, persist a visible failure outcome on the
run row. Silent swallow is the bug class this program exists to kill.
