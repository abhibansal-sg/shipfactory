- Learning objective: see what each phase produces, where rework returns, and why the approval step remains exclusively human.
- Data structure: a left-to-right 12-step progression with three nested phase bands and four backward rework loops.
- Visual priority: workflow first; trust-boundary callouts second; evidence/test status third.
- Tone: precise engineering schematic, public-repo appropriate, no competitor names or private paths.
- Language: English.
- Recommended treatment: technical-schematic + linear-progression, landscape 16:9.
- Accessibility: high contrast, restrained palette, large labels, no tiny paragraphs, distinct human gate color, avoid red/green-only state encoding.
